触发器模块设计说明书（Clawith / Agent Triggers）
1. 概述与目标
1.1 定位
触发器模块为数字员工（Agent）提供自管理的唤醒条件：在指定时间、周期、外部事件或消息到达时，系统自动拉起该 Agent 的推理流程，并把「为何被唤醒」作为上下文交给模型。

在产品语境中，该能力与 Aware（觉察）、Focus（焦点任务）、内心独白（Reflection，来源为 source_channel=trigger） 联动。

1.2 设计目标
目标	说明
统一调度
用单一后台循环替代分散的定时/轮询逻辑入口（与 Heartbeat 协同，见 §9）
Agent 自治
通过工具 set_trigger / update_trigger / cancel_trigger / list_triggers 由模型自行管理触发器
人机协同
管理端 REST 可查看/补丁/删除；Webhook 供外部系统推送事件
可控成本
每 Agent 触发器数量上限、冷却、最大触发次数、Webhook 速率等

4. 触发器类型与配置契约
4.1 cron
config：expr（cron 表达式），可选 timezone 覆盖。
评估：croniter；基准时间为 last_fired_at 或 created_at；时区优先 config，否则 get_agent_timezone(agent_id)。
说明：在 Agent 所在时区解释「当前是否已过下一次应触发点」。
4.2 once
config：at（ISO-8601 时间字符串）。
条件：now >= at 且 fire_count == 0。
触发后：预更新阶段将 is_enabled=False（一次性）。
4.3 interval
config：minutes。
条件：(now - (last_fired_at or created_at)) >= timedelta(minutes)。
4.4 poll（HTTP 轮询 + 变更检测）
config：url 必填；method、headers、json_path（默认 $）、fire_on：change | match、match_value、interval_min（与 MIN_POLL_INTERVAL_MINUTES 取 max，默认至少 5 分钟）。
逻辑：间隔内不请求；请求后对比 config._last_value（持久化在 JSONB），change 模式下首次只建基线不触发。
安全：_is_private_url 防止 SSRF（内网/localhost 等拒绝）。
4.5 on_message
config：二选一 — from_agent_name（跨 Agent）或 from_user_name（飞书/Slack/Discord 人类用户）。
辅助：创建时可写入 _since_ts，避免把历史消息误判为新消息。
评估：查 ChatMessage + ChatSession，匹配后发件方与时间窗口；匹配内容写入 config._matched_message / _matched_from 供上下文使用。
4.6 webhook
创建：set_trigger 自动生成 config.token；对外 URL：{PUBLIC_URL}/api/webhooks/t/{token}（PUBLIC_URL 缺失时有回退域名）。
投递：POST /api/webhooks/t/{token} 将 body 截断写入 _webhook_payload，并设 _webhook_pending=True。
评估：Daemon 中若 _webhook_pending 为真则视为应触发；触发后清空 pending/payload。
安全：可选 HMAC（config.secret 与 x-hub-signature-256）；按 token + Agent 的 webhook_rate_limit 限流。
5. 运行时：Trigger Daemon（trigger_daemon.py）
5.1 生命周期
在 main.py 的 lifespan 中与 Feishu WS 等一并 asyncio.create_task(start_trigger_daemon()) 启动。
主循环：约每 TICK_INTERVAL = 15 秒 执行一次 _tick()，循环末尾 asyncio.sleep(15)。
5.2 单次 Tick 流程
加载 is_enabled=True 的全部 AgentTrigger。
对每条触发器调用 _evaluate_trigger(trigger, now)；过期则禁用并跳过。
将本 tick 应触发 的触发器按 agent_id 分组。
去重：同一 Agent 在 DEDUP_WINDOW = 30 秒 内不重复唤起（进程内 _last_invoke）。
关键：先写库再调 LLM：对命中的触发器立即更新 last_fired_at、fire_count，once 禁用，webhook 清理 pending，max_fires 用尽则禁用——避免长推理期间下一 tick 重复触发。
asyncio.create_task(_invoke_agent_for_triggers(...)) 异步执行，不阻塞 tick。
5.3 与 Heartbeat 的关系
每 4 个 tick（约 60 秒）调用一次 heartbeat._heartbeat_tick()，使周期性「环境觉察」与触发器共用同一进程内的调度节奏（日志中「heartbeat every ~60s」即此意）。
6. 触发后的执行链：_invoke_agent_for_triggers
校验 Agent 存在且未过期；必须有 primary_model_id 且模型 enabled。
组装 中文触发上下文（触发器名、类型、reason、focus_ref、on_message 摘要、webhook payload 截断等）。
创建 ChatSession：source_channel="trigger"，标题如「内心独白：…」。
写入一条 user 角色的 ChatMessage（内容为上述上下文）。
调用 call_llm（与主对话相同管线），支持 on_tool_call 回调，工具调用写入同一会话便于 Reflection 页展示。
将助手回复写入会话；并尝试 推送到用户当前 WebSocket 会话（trigger_notification），否则落到最近 Web 会话。
write_audit_log("trigger_fired", ...)。

8. HTTP API
8.1 管理端（需登录）— app/api/triggers.py
GET /api/agents/{agent_id}/triggers：列表，含 config 全量（含 token 等敏感信息——管理端需注意权限）。
PATCH /api/agents/{agent_id}/triggers/{trigger_id}：部分更新 config、reason、开关、max_fires、cooldown、expires_at。
DELETE ...：物理删除。
前缀挂载在 /api/agents 下（与 main.py 中 include_router 一致）。

8.2 公开 Webhook — app/api/webhooks.py
POST /api/webhooks/t/{token}：无 Bearer；依赖 token 保密、可选 HMAC、速率与 body 大小限制。

10. 前端（Aware 页）
triggerApi.list / update / delete（frontend/src/services/api.ts）对接管理 API。
Agent 详情 → Aware：展示触发器列表、focus.md、仅 source_channel===trigger' 的会话作为「内心独白」，定时刷新。
11. 与系统其他模块的衔接
系统提示词（agent_context.py）：注入「活跃触发器」列表、时区说明，以及 Focus / trigger 写作规范（reason 怎么写、focus_ref 绑定等）。
WebSocket：触发完成后下发 type: "trigger_notification"，便于前端展示「触发器唤醒」气泡。
12. 边界、风险与改进方向（基于当前实现）
Daemon 单进程内存去重：_last_invoke、Webhook 内存限流在多进程/多实例部署下不共享，可能重复唤起或限流失效——若水平扩展需 Redis 等协调。
on_message 用户侧：名称模糊匹配、无租户强约束时可能误匹配——需结合业务加租户/会话隔离。
管理 API 列出完整 config：含 webhook token，仅应管理员或创建者可见（当前依赖路由层认证，细粒度 RBAC 可再加强）。
tool_seeder 中触发器类型的 enum 可能略旧于 agent_tools（例如缺 webhook）——以运行时 VALID_TRIGGER_TYPES 为准，种子与文档宜对齐。
13. 小结
触发器模块以 AgentTrigger 为唯一事实来源，由 trigger_daemon 15s 一轮统一评估时间类/轮询/消息/Webhook 类条件；命中后先落库再异步调用 LLM，并通过 Reflection 会话 + WebSocket 完成可观测与用户触达。工具 API 负责 Agent 自治创建与运维，REST 与 Webhook 分别服务控制台与外部集成，形成完整闭环。
